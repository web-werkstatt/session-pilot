How the Quality Scanner works

1
Scan

Choose a project and click "Scan". The scanner checks file sizes, code duplication, complexity, CSS/JS quality and architecture rules.

2
Set baseline

Freeze the current state. From now on "Diff" only shows new or resolved issues — no noise from legacy problems.

3
Improve

Work through the issues: consolidate duplicates, split complex functions, break up large files. Always small steps, 1-3 files per run.

4
Check diff

After each change: Scan + Diff. The score must not drop. If it rises: set a new baseline.
